Methodology

Four Forecasting Approaches

📈
ARIMA(1,1,1)
Autoregressive Integrated Moving Average. Three variants tested: (0,0,0) baseline, (1,0,1), and (1,1,1). Walk-forward applied to best order via 258 sequential refits (~1–2 min on Colab).
Statistical Walk-Forward MAPE 1.62%
🌊
SARIMA(2,1,2)(1,1,1)₅
Seasonal ARIMA with s=5 (weekly trading cycle). Order selected via AIC grid search over p,q∈{0,1,2}, P,Q∈{0,1}. Best AIC = 4575.36. Walk-forward: 258 refits (~3–5 min).
Seasonal AIC Grid Search MAPE 1.67%
🔮
Facebook Prophet
Additive decomposition: trend g(t) + seasonality s(t) + holidays h(t) + noise. Fitted on training set only (no data leakage). Yearly, weekly, daily seasonality; changepoint prior 0.05.
Additive Changepoint No Walk-Forward
🧠
Stacked LSTM (64-64-32)
3-layer LSTM with 60-day sliding window. 62,369 trainable parameters. Dropout 0.2 each layer; Adam optimizer; early stopping patience=10; MinMax scaling; random seed 42.
Deep Learning 60-day Window MAPE 3.60%
07
Key Innovation

Walk-Forward Evaluation Protocol

1
Initialise History H
Start with Ntrain = 1,000 observations in the history buffer. All training data is available.
2
Fit ARIMA(1,1,1) on H
For each step t = 1…258, fit the model on the current history H. Model re-estimates parameters at every step.
3
Forecast One Step Ahead
Generate ŷt for the next trading day only (1-step-ahead forecast).
4
Append Actual yt to H
After observing the actual price, append it to H. The model anchors on the real price at each step — eliminating drift.
5
Evaluate {ŷt} against {yt}
Compute all 9 metrics on the collected 258 forecasts. Cost: ~1–2 min for ARIMA, ~3–5 min for SARIMA.
🚀

Why It Works

In real-world trading, actual prices are observed before each new trading day. Walk-forward simulates this exactly — the model never predicts far beyond its last known price.

ARIMA Equation
φ(B)(1−B)ᵈYₜ = θ(B)εₜ
φ(B) = autoregressive polynomial, θ(B) = moving average polynomial, B = backshift operator, εₜ ~ N(0,σ²)
SARIMA Equation
Φ(Bˢ)φ(B)(1−B)ᵈ(1−Bˢ)ᴰYₜ = Θ(Bˢ)θ(B)εₜ
s=5 (5 trading days/week). Seasonal order (2,1,2)(1,1,1)₅ selected by AIC grid search.
Static vs Walk-Forward
  • Static: fit once, forecast all 258 days → catastrophic drift
  • Static ARIMA MAPE: 21.96%, R² = −1.83
  • Walk-Forward ARIMA MAPE: 1.62%, R² = 0.9786
  • Improvement: 91% MAPE reduction

Our Position in the Literature

We extend prior work by introducing walk-forward evaluation to ARIMA/SARIMA on equity data, demonstrating a 91% MAPE reduction unavailable in any prior study. Fischer & Krauss (2018) inspired the rolling protocol; Kwarteng et al. (2024) serve as the primary benchmark.

Discussion

Key Insights & Limitations

Why Walk-Forward ARIMA Wins
  • AAPL follows near-random-walk: ARIMA(1,1,1) with d=1 is optimal for short-horizon
  • Walk-forward anchors on actual price at each step → eliminates structural break drift
  • Slight edge over SARIMA WF (1.62% vs 1.67%) — weekly seasonality adds marginal value once drift is eliminated
  • Near-zero bias: MBE = −$0.52/day
LSTM Third Despite Deep Learning
  • MAPE 3.60%, R² = 0.9265 — strong but third
  • Cannot update on actual test prices step by step (asymmetry)
  • MBE = −$4.66 (consistent underestimation during COVID surge)
  • Early stopping confirmed good generalisation (Fig 12)
  • Walk-forward LSTM is future work (prohibitive: 258 retrains)
Limitations
  • Univariate only — no volume, RSI, MACD, sentiment
  • Single train/test window — multi-window CV needed for robustness
  • Prophet calendar/trading-day mismatch unresolved
  • Walk-forward LSTM not attempted (computationally expensive)
  • Results specific to AAPL 2015–2020 period

Conclusions

Main Findings & Future Work

✓
Evaluation protocol determines model ranking
Walk-forward reduced ARIMA MAPE 21.96% → 1.62% — a 91% improvement from evaluation strategy alone, with no change in model order.
✓
Walk-forward ARIMA(1,1,1) is best-performing
MAPE 1.62%, R² = 0.9786, MAE $4.12, MBE −$0.52, Theil's U₂ = 0.013. Reflects informational advantage of anchoring on actual prices.
✓
LSTM ranks third (MAPE 3.60%, R² = 0.9265)
Deep learning does not automatically outperform simpler methods on near-random-walk financial series.
✓
All models beat the naïve baseline
Theil's U₂ < 1 for all seven evaluated configurations. Even static models provide some value over persistence forecasting.
Future Work
  • Add exogenous variables: volume, RSI, MACD via SARIMAX or multivariate LSTM
  • Apply walk-forward symmetrically to LSTM and Prophet for fully fair comparison
  • Explore hybrid ARIMA-LSTM residual correction
  • Multi-window rolling cross-validation for more robust estimates
  • Fix Prophet calendar/trading-day alignment (filter to 258 trading days)
🏆

Central Lesson

For financial time series, correct evaluation protocol is a first-order concern — more impactful than model selection. A walk-forward ARIMA(1,1,1) with MAPE 1.62% outperforms a complex LSTM with MAPE 3.60%.
